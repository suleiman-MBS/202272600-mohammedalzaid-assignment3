## 1. Project Overview & Tools
This report documents the use of AI-assisted development across two phases. Phase 1 focused on structural setup and responsive design, while Phase 2 focused on JavaScript interactivity and user state management.

## 2. Phase 1: Foundation & Repository Setup (Assignment 1)
During the initial setup, I used AI primarily as a technical tutor to navigate the command line and Git environment.

### Key Areas of Assistance:
* **Repository Setup:** Guided me through the process of cloning a remote GitHub repository to my local KFUPM environment using the Command Prompt.
* **Git Troubleshooting:** Explained the "Staging" vs. "Committing" lifecycle and helped resolve "Please tell me who you are" errors by configuring `user.name` and `user.email`.
* **Semantic Structure:** Assisted in generating the initial HTML5 boilerplate and the CSS Grid structure to ensure the site was mobile-responsive from day one.
* **Content Refinement:** Helped structure my raw resume data (GPA 3.5+, Oracle Ambassador, KAUST Trainee) into professional Markdown for the README.

### Phase 1 Example Prompts:
* *"How to get clone with my computer?"*
* *"Why I can not commit? it says log in to git..."*

## 3. Phase 2: Interactivity & State Management (Assignment 2)
In the second phase, I transitioned from using AI for setup to using it for advanced JavaScript logic and modern UI/UX principles.

### Key Areas of Assistance:
* **Dynamic Content Logic:** AI assisted in utilizing the `Date()` object in JavaScript to inject a time-sensitive greeting ("Good Morning/Afternoon/Evening") into the DOM, fulfilling the interactivity requirement.
* **Persistent Theme Engine:** Guided the implementation of the **Web Storage API** using `localStorage`. This ensures the user's "Dark Mode" preference remains active even after a page refresh.
* **CSS Variable Architecture:** AI suggested refactoring static CSS into a variable-based system (`:root`). This made the theme-switching logic cleaner and more scalable for future assignments.
* **Micro-Animations:** AI provided the CSS `transform` and `transition` properties required to create professional "lift" and "glow" effects on my research project cards.
* **Enhanced Form Feedback:** Assisted in replacing standard browser alerts with an in-page **Form Validation** system, providing immediate visual feedback within the UI.

### Phase 2 Example Prompts:
* *"How do I make a dark mode button that remembers the setting after refresh?"*
* *"Help me write a JS function that changes text based on the time of day."*
* *"Suggest a professional hover effect for a portfolio card using transitions."*

## 4. Benefits & Challenges
* **Benefits:** The AI acted as a technical tutor, explaining *why* errors were happening (like the DOM Lifecycle) rather than just fixing them. This ensured my scripts only run after the HTML is fully loaded.
* **Challenges:** Merging AI-suggested Dark Mode logic with my existing Assignment 1 styles caused initial layout shifts. I had to manually refactor my CSS and adjust the z-index to ensure the toggle button functioned correctly.

## 5. Learning Outcomes
Through this two-phase AI-assisted process, I learned:
1. **Git Lifecycle:** Linking local folders to remote repositories and the importance of `git config`.
2. **State Management:** Using `localStorage` to handle persistent user preferences.
3. **DOM Manipulation:** Targeting specific IDs to update content dynamically without page reloads.
4. **Professional UI/UX:** The power of CSS Variables and smooth transitions in building professional-grade web applications.
